    **classifySet** — Pure function (per D-01). Priority order per D-02: PR > +1 > Matched.
    - PR per D-03: exact weight x reps never done before. Require hasCompletedHistory=true to avoid Pitfall 1.
    - +1 per D-04: beat last session's same set number (more reps at same weight OR same reps at higher weight).
    - Matched per D-05: identical weight and reps as last session's same set number.
    - Return null if none match. Volume Up and Comeback are exercise-level, not per-set.

    **suggestTarget** — Pure function (per D-15 through D-18). Format per D-16: "Last time: {weight} x {reps}. Try {weight} x {reps+1} or {weight+5} x {reps}". Return null if no history.

    **getSetBadgesForExercise** — Async function that queries DB.
    1. Get exercise record to find exerciseName
    2. Find all workoutExercise records for this exerciseName
    3. Get parent workouts, filter to completed (use .filter() for completedAt, NOT .where() — Dexie null index pitfall), exclude currentWorkoutId per D-09
    4. Sort by completedAt descending, take most recent
    5. Get sets from that workout's exercise
    6. Get ALL historical sets for this exercise (for PR detection)
    7. Check hasCompletedHistory (at least one completed workout with this exercise)
    8. For each current set, call classifySet
    9. Check isComeback: exercise not in last completed workout but appears in an older one (per D-07)
    10. Return Map<setId, BadgeType> and isComeback flag

    **generateWorkoutSummary** — Async function per D-20 through D-26.
    1. Get all exercises for workoutId
    2. For each exercise, compute current volume (sum weight x reps)
    3. Find previous completed workout with same NAME (per D-21), get its volume
    4. Per-exercise: find previous session for each exercise name, compute direction (up/down/same/new per D-22)
    5. Win count per D-23: count exercises with direction 'up'
    6. Best achievement per D-24: prefer PRs (from set badges), fallback to biggest volume % gain
    7. If no previous same-name workout exists per D-26: previousTotalVolume=null, volumeChangePercent=null, congratulatory bestAchievement

    **Test setup**: Follow workoutService.test.ts pattern — beforeEach clears all tables. Create helper functions to seed test data (create workout, add exercise, log sets, finish workout).

    **IMPORTANT ordering for generateWorkoutSummary**: This function is called BEFORE finishWorkout (per Pitfall 6 in RESEARCH.md). The current workout has no completedAt, so filtering completed workouts naturally excludes it.
